# Licensing and Attribution

TDFam is licensed under the Apache License, Version 2.0. See the root
`LICENSE` file for the full license text and the root `NOTICE` file for
project attribution notices.

This page explains the intended licensing boundary for people using TDFam to
build and distribute TouchDesigner operator families.

## What the TDFam License Covers

The Apache-2.0 license covers TDFam itself, including:

- the TDFam core framework
- the TD Registry
- menu integration code
- manifest, stubbing, update, callback, and configuration tools
- examples, templates, documentation, and support files distributed with this
  project

If you distribute TDFam, a modified version of TDFam, or a TouchDesigner
component or operator family that includes TDFam, preserve the Apache-2.0
license notices and the `NOTICE` attribution file as required by the license.

## User-Created Families

Operator families, `.tox` files, manifests, callbacks, artwork, and
project-specific logic created by TDFam users may use their own licenses.

The TDFam parts remain Apache-2.0. Your own family-specific work can be open
source, commercial, proprietary, experimental, internal, or released under a
different license.

## Attribution

If you distribute a TouchDesigner family, `.tox`, project, or toolkit that
includes TDFam, please include a visible attribution in your project
documentation, README, About panel, Help DAT, credits section, or third-party
notices.

Suggested attribution:

> Built with TDFam, an open-source TouchDesigner operator-family framework
> created by Lyell Hintz / dotsimulate and Dan Molnar / function.str.

Short attribution:

> Built with TDFam by dotsimulate and function.str.

Project links:

- dotsimulate: <https://dotsimulate.com>
- function.str: <https://www.functionstore.xyz/link-in-bio>

This attribution does not need to appear in every user interface, render,
performance, exported work, or marketing material. A clear credit in the
distributed package or project documentation is enough.
